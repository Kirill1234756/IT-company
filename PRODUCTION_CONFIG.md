# 🔧 Настройка фронтенда для продакшена

## 📋 Что нужно изменить для api.kodifyweb.ru

Для продакшена нужно указать URL API бекенда. Есть два способа:

## Вариант 1: Через переменную окружения (рекомендуется)

### Шаг 1: Создать `.env` файл

В корне папки `frontend` создайте файл `.env`:

```bash
cd frontend
cp .env.example .env
nano .env
```

### Шаг 2: Указать API URL для продакшена

В файле `.env` укажите:

```env
# Для продакшена
VITE_API_URL=https://api.kodifyweb.ru/api
```

**Важно:** 
- Используйте `https://` (не `http://`)
- URL должен заканчиваться на `/api`
- После изменения `.env` нужно пересобрать фронтенд

### Шаг 3: Пересобрать фронтенд

```bash
npm run build
```

---

## Вариант 2: Через index.html (альтернативный)

Если вы не можете использовать `.env` файл при сборке, можно задать URL прямо в `index.html`.

### Отредактируйте `frontend/index.html`

Найдите строку:
```javascript
window.API_BASE = 'http://localhost:3000/api'
```

Замените на:
```javascript
window.API_BASE = 'https://api.kodifyweb.ru/api'
```

**Недостаток:** это значение будет жестко зашито в сборке.

---

## ✅ Проверка

После настройки проверьте:

1. **Соберите проект:**
   ```bash
   npm run build
   ```

2. **Проверьте, что в сборке используется правильный URL:**
   ```bash
   # Посмотрите в dist/index.html или dist/assets/*.js
   grep -r "api.kodifyweb.ru" dist/
   ```

3. **Проверьте в браузере:**
   - Откройте DevTools (F12)
   - Перейдите на вкладку Network
   - Отправьте форму
   - Убедитесь, что запросы идут на `https://api.kodifyweb.ru/api`

---

## 🔍 Где используется API URL

API URL используется в следующих файлах:
- `src/api/contact-form.ts` - контактная форма
- `src/api/client-form.ts` - форма "Стать клиентом"
- `src/api/calculator-form.ts` - калькулятор стоимости

Все эти файлы автоматически читают `VITE_API_URL` из переменных окружения.

---

## 📝 Пример полной настройки для продакшена

```bash
# 1. Перейти в папку фронтенда
cd frontend

# 2. Создать .env файл
cat > .env << EOF
VITE_API_URL=https://api.kodifyweb.ru/api
EOF

# 3. Установить зависимости (если еще не установлены)
npm install

# 4. Собрать проект
npm run build

# 5. Проверить сборку
ls -la dist/
```

---

## ⚠️ Важно помнить

1. **CORS:** Убедитесь, что в бекенде (`FRONTEND_URL`) указан домен фронтенда `https://kodifyweb.ru`
2. **HTTPS:** Используйте только HTTPS для продакшена
3. **Пересборка:** После изменения `.env` всегда пересобирайте проект (`npm run build`)
4. **Проверка:** Проверяйте в Network tab браузера, что запросы идут на правильный адрес

---

## 🔄 Обновление после изменения

Если вы изменили API URL в `.env`:

```bash
cd frontend
npm run build
# Затем загрузите новую сборку на сервер
```

Если фронтенд развернут через Docker или другой CI/CD, убедитесь, что `.env` файл доступен при сборке.
