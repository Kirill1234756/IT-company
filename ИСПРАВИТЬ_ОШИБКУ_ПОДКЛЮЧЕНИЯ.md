# 🔧 Исправление ошибки подключения к серверу

## Проблема

Фронтенд показывает: "Ошибка подключения к серверу. Проверьте интернет-соединение и попробуйте еще раз."

## Причина

Фронтенд не может подключиться к API `https://api.kodifyweb.ru/api` потому что:

1. Фронтенд не пересобран с правильным API URL
2. На reg.ru хостинге старая сборка с неправильным URL

## ✅ Решение

### Шаг 1: Пересобрать фронтенд

На вашем локальном компьютере:

```bash
cd C:\Users\user\Desktop\it-company\frontend
npm run build
```

Это создаст обновленную сборку в папке `frontend/dist/` с правильным API URL (`https://api.kodifyweb.ru/api`).

### Шаг 2: Загрузить обновленную сборку на reg.ru

1. Зайдите в панель управления reg.ru
2. Откройте файловый менеджер для домена `kodifyweb.ru`
3. Загрузите **все файлы** из папки `frontend/dist/` в корневую директорию сайта
4. Убедитесь, что старые файлы заменены

**Важно:** Загрузите всю папку `dist/`, включая:

- `index.html`
- Папку `assets/` со всеми JS и CSS файлами
- Все остальные файлы

### Шаг 3: Проверить настройки бекенда (опционально)

На VPS убедитесь, что бекенд перезапущен после изменений:

```bash
cd ~/projects/kodify/backend
docker-compose restart backend
```

### Шаг 4: Проверить работу

1. Откройте `https://kodifyweb.ru/contacts`
2. Попробуйте отправить форму
3. Ошибка должна исчезнуть

## 🔍 Проверка правильности сборки

После сборки проверьте, что правильный URL используется:

**Windows (PowerShell):**

```powershell
Select-String -Path "dist/assets/*.js" -Pattern "api.kodifyweb.ru" | Select-Object -First 3
```

**Linux/Mac:**

```bash
grep -r "api.kodifyweb.ru" dist/
```

Должно найти упоминание `api.kodifyweb.ru` в собранных файлах.

## ⚠️ Важные замечания

1. **После каждого изменения `index.html` или `.env` нужно пересобрать фронтенд**
2. **Vite заменяет переменные окружения во время сборки** - просто изменить `.env` на сервере не поможет
3. **Файлы из `dist/` нужно загружать на reg.ru хостинг** - это статичные файлы
