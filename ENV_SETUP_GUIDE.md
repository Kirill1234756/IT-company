# 🔧 Как работает `.env` во фронтенде с Vite

## 📋 Как Vite работает с переменными окружения

**Vite автоматически подтягивает переменные из `.env` файла при сборке проекта.**

### Ключевые моменты:

1. **Переменные должны начинаться с `VITE_`**
   - ✅ Правильно: `VITE_API_URL`
   - ❌ Неправильно: `API_URL` (не будет доступна)

2. **Vite автоматически читает `.env` файл**
   - При запуске `npm run dev` - читает `.env`
   - При запуске `npm run build` - читает `.env` и встраивает значения в код

3. **Доступ через `import.meta.env`**
   - В коде: `import.meta.env.VITE_API_URL`
   - Vite заменяет это на реальное значение при сборке

---

## 📁 Где создать `.env` файл

Создайте файл `.env` в корне папки `frontend`:

```
it-company/
├── frontend/
│   ├── .env          ← ЗДЕСЬ
│   ├── .env.example
│   ├── package.json
│   ├── vite.config.ts
│   └── src/
└── backend/
```

## 🚀 Пошаговая инструкция

### Вариант 1: На локальном компьютере (Windows)

1. **Откройте папку `frontend`** в редакторе кода или файловом менеджере

2. **Создайте файл `.env`** в корне папки `frontend`

3. **Добавьте в файл:**
   ```env
   VITE_API_URL=https://api.kodifyweb.ru/api
   ```

4. **Сохраните файл**

5. **Пересоберите проект:**
   ```bash
   cd frontend
   npm run build
   ```

---

### Вариант 2: На сервере (через SSH/VNC)

1. **Подключитесь к серверу** (через SSH или VNC)

2. **Перейдите в папку фронтенда:**
   ```bash
   cd ~/projects/it-company/frontend
   # или где у вас лежит фронтенд
   ```

3. **Создайте `.env` файл:**
   ```bash
   nano .env
   ```

4. **Вставьте содержимое:**
   ```env
   VITE_API_URL=https://api.kodifyweb.ru/api
   ```

5. **Сохраните:**
   - `Ctrl+O` (сохранить)
   - `Enter` (подтвердить имя файла)
   - `Ctrl+X` (выйти)

6. **Пересоберите проект:**
   ```bash
   npm run build
   ```

## 🔍 Как проверить, что все работает

### Способ 1: Проверить сборку

После сборки проверьте, что URL встроен в код:

```bash
cd frontend
npm run build

# На Windows PowerShell:
Select-String -Path "dist/assets/*.js" -Pattern "api.kodifyweb.ru"

# На Linux/Mac:
grep -r "api.kodifyweb.ru" dist/
```

Должно найти упоминание `api.kodifyweb.ru` в собранных файлах.

---

### Способ 2: Проверить в браузере

1. **Откройте сайт** `https://kodifyweb.ru`

2. **Откройте DevTools** (F12)

3. **Перейдите на вкладку Network**

4. **Отправьте любую форму** (контактная форма, калькулятор и т.д.)

5. **Проверьте запрос:**
   - Должен идти на `https://api.kodifyweb.ru/api/...`
   - Не должен идти на `http://localhost:3000/api`

---

### Способ 3: Проверить в консоли браузера

В DevTools → Console выполните:

```javascript
// Проверить, что переменная доступна (только в dev режиме)
console.log(import.meta.env.VITE_API_URL)
```

**Примечание:** В production сборке переменные встроены в код, так что прямо проверить через консоль не получится. Но запросы должны идти на правильный адрес.

## 📝 Примеры `.env` файлов

### Для разработки (локально):

```env
# API URL для локального бекенда
VITE_API_URL=http://localhost:3000/api
```

### Для продакшена:

```env
# API URL для production бекенда
VITE_API_URL=https://api.kodifyweb.ru/api
```

### Можно использовать несколько файлов:

- `.env` - общие настройки (не коммитится в Git)
- `.env.local` - локальные настройки (переопределяет `.env`, не коммитится)
- `.env.production` - для production сборки (`npm run build`)
- `.env.development` - для dev режима (`npm run dev`)

**Приоритет:** `.env.local` > `.env.production/.env.development` > `.env`

---

## ⚠️ Важные моменты

1. **Файл `.env` не должен быть в Git**
   - Убедитесь, что `.env` в `.gitignore`
   - Можно коммитить `.env.example` как шаблон

2. **Переменные встраиваются при сборке**
   - После изменения `.env` **обязательно** пересоберите: `npm run build`
   - Изменения не применятся автоматически без пересборки

3. **Только переменные с `VITE_` доступны**
   - Vite специально ограничивает доступ к переменным для безопасности
   - Только `VITE_*` переменные доступны в коде

4. **В production переменные заменяются на значения**
   - В собранном коде вы не увидите `import.meta.env.VITE_API_URL`
   - Вместо этого будет реальное значение: `"https://api.kodifyweb.ru/api"`

---

## 🔄 Что происходит при сборке

```
1. Вы запускаете: npm run build
   ↓
2. Vite читает .env файл
   ↓
3. Vite находит VITE_API_URL=https://api.kodifyweb.ru/api
   ↓
4. В коде заменяет import.meta.env.VITE_API_URL на "https://api.kodifyweb.ru/api"
   ↓
5. Собирает оптимизированный код в папку dist/
   ↓
6. Готово! В dist/ код уже содержит правильный URL
```

## ❓ Часто задаваемые вопросы

### Q: Нужно ли что-то настраивать дополнительно?

**A:** Нет! Vite автоматически читает `.env` файл. Просто создайте файл и пересоберите проект.

---

### Q: Можно ли использовать разные `.env` для dev и production?

**A:** Да! Создайте:
- `.env.development` - для `npm run dev`
- `.env.production` - для `npm run build`

---

### Q: Почему переменная не работает?

**A:** Проверьте:
1. Начинается ли переменная с `VITE_`?
2. Находится ли `.env` в корне папки `frontend`?
3. Пересобрали ли вы проект после изменения?

---

### Q: Где хранить `.env` на сервере?

**A:** В той же папке, где код фронтенда. При сборке Vite автоматически прочитает его.
